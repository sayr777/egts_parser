# Рекомендации и возможные улучшения EGTS Parser

**Дата:** 2026-06-11

## 1. Тесты
- Добавить полноценный pytest suite.
- Edge-кейсы: длинные пакеты, routed header, CRC-ошибки, неполные данные.
- Реальные тестовые пакеты из БНСО (РНИС).

## 2. Производительность
- Для high-load ingestion (РНИС) — оптимизировать hot-path или мигрировать на Rust/Go.
- Бенчмарки decode/encode.

## 3. Интеграции
- n8n-ноды для EGTS.
- Примеры загрузки в PostGIS / ClickHouse.
- Валидация пакетов по ГОСТ и бизнес-правилам РНИС.
- Интеграция с существующими Python-сервисами (MoySklad, Flora и т.д.).

## 4. Документация
- Полные примеры для ПМИ / И3.1 / формуляров.
- C4/Mermaid-диаграммы архитектуры.
- OpenAPI/Swagger для Cloud Function.
- Примеры использования в К8s / Docker.

## 5. Мобильное приложение
- Background service / foreground tracking.
- Оффлайн-очередь пакетов.
- Поддержка большего количества сенсоров.

## 6. Общее
- Добавить LICENSE (MIT рекомендуется).
- GitHub Actions (tests + build APK).
- Поддержка EGTS over WebSocket / MQTT.
- Генератор тестовых пакетов на основе шаблонов.
