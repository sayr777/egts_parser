# Глубокий анализ стандартов RTLS и рекомендации для EGTS Parser

**Дата:** 2026-06-11  
**Автор:** Grok (для проекта sayr777/egts_parser)  
**Цель:** Полный разбор международных и российских стандартов RTLS с привязкой к вашему EGTS-расширению.

## 1. Основной стандарт: ISO/IEC 24730 (серия)

**ISO/IEC 24730** — "Information technology — Real-time locating systems (RTLS)".

### Ключевые части:

| Часть | Год | Название | Основное содержание | Технологии |
|-------|-----|----------|---------------------|------------|
| **24730-1** | 2014 | Application Programming Interface (API) | Единый API для приложений (location queries, tag management, zones) | Независим от PHY |
| **24730-2** | 2012 | DSSS 2.4 GHz | Air interface на базе Direct Sequence Spread Spectrum | 2.4 GHz, DBPSK/QPSK |
| **24730-5** | 2010 | CSS 2.4 GHz | Chirp Spread Spectrum — высокая устойчивость к помехам | Chirp modulation |
| **24730-61** | 2013 | UWB PHY/MAC Low Rate PRF | Ultra-Wide Band, низкая частота импульсов | IEEE 802.15.4a based |
| **24730-62** | 2013 | UWB PHY/MAC High Rate PRF | Высокоточное позиционирование (10-30 см) | HRP UWB |

**Российские версии (ГОСТ Р):**
- ГОСТ Р ИСО/МЭК 24730-1-2017 (API)
- ГОСТ Р ИСО/МЭК 24730-62-2018 и другие гармонизированные версии.

## 2. Связанные стандарты

- **IEEE 802.15.4-2020** + **802.15.4z-2020** — базовый PHY/MAC для UWB RTLS (улучшенный ranging, security, integrity).
- **ISO/IEC 18000** — RFID, частично пересекается.
- **BLE 5.1+** (AoA/AoD) — не входит в ISO 24730 напрямую, но широко используется в RTLS.
- **Wi-Fi FTM** (802.11-2016) — для Wi-Fi based RTLS.

## 3. Сравнение с вашим EGTS RTLS (SRT 200–203)

| Аспект | ISO/IEC 24730 | Ваш EGTS RTLS | Рекомендация |
|--------|---------------|---------------|--------------|
| **API** | Чётко определён в Part 1 | Кастомный через EGTS TELEDATA | Добавить соответствие 24730-1 |
| **Air Interface** | DSSS/CSS/UWB | BLE/WiFi/NFC/UWB (через мобильное app) | Использовать стандартные PHY где возможно |
| **Точность** | До 10-30 см (UWB) | Зависит от hardware | Добавить поля quality, zone_id, rssi |
| **Транспорт** | HTTP/SLMP и др. | EGTS (уже в РНИС) | Сохранить EGTS как транспорт |
| **Интероперабельность** | Высокая между вендорами | Высокая внутри EGTS-экосистемы | Гибридный подход |

## 4. Конкретные рекомендации по реализации

1. **Документация**:
   - Везде явно указывать соответствие **ISO/IEC 24730-1 (API)** + ссылки на **ГОСТ Р ИСО/МЭК 24730-1-2017**.

2. **UWB-расширения**:
   - Рассмотреть поддержку **IEEE 802.15.4z-2020** для ranging (ToF, TDoA, Phase Difference).
   - Добавить новые SRT или поля в существующие 200-203.

3. **Модели и Excel-парсер**:
   - Добавить стандартные RTLS-метрики: `quality` (0-100), `zone_id`, `tag_type`, `rssi`, `battery_level`, `confidence_level`.

4. **ТЗ RTLS v2**:
   - Добавить раздел **«Соответствие международным стандартам»** с таблицами выше.

5. **Тестирование**:
   - Генерировать тестовые пакеты, соответствующие требованиям стандарта к частоте обновлений и точности.
