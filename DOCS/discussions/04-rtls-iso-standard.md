# RTLS как стандарт: ISO/IEC 24730 и интеграция с EGTS

**Дата:** 2026-06-11  
**Автор:** Grok (анализ для egts_parser)

## Обзор стандарта ISO/IEC 24730

**ISO/IEC 24730** — международный стандарт **Information technology — Real-time locating systems (RTLS)**. Он определяет:

- **Единый Application Programming Interface (API)** — Part 1.
- Несколько **протоколов радиоинтерфейса (Air Interface Protocols)** для разных технологий.

### Основные части стандарта (актуальные на 2026):

| Часть | Название | Основное содержание |
|-------|----------|---------------------|
| **24730-1:2014** | Application programming interface (API) | Единый API для приложений: получение координат, состояние тегов, управление инфраструктурой. Поддержка SLMP, HTTP. |
| **24730-2:2012 (+21, +22)** | Direct Sequence Spread Spectrum (DSSS) 2.4 GHz | DSSS на 2.4 ГГц, различные схемы кодирования (DBPSK, QPSK, WOQPSK). |
| **24730-5:2010** | Chirp Spread Spectrum (CSS) 2.4 GHz | CSS (chirp) — хорошая дальность и устойчивость к помехам. |
| **24730-61 / 62:2013** | Ultra Wide Band (UWB) | Высокоточное позиционирование (десятки см) с низкой/высокой частотой повторения импульсов. |

**Цель стандарта**: Обеспечить **интероперабельность** оборудования разных производителей для систем позиционирования в реальном времени (asset tracking, indoor navigation).

В России приняты гармонизированные версии: **ГОСТ Р ИСО/МЭК 24730-1-2017** и др.

## Связь с EGTS (ваш проект)

Ваш вариант RTLS в `egts_parser` — **кастомное vendor-specific расширение**:

- **SRT 200–203** внутри сервиса `EGTS_TELEDATA_SERVICE`.
- Не противоречит ГОСТ Р 54619-2011, т.к. SRT > 127 зарезервированы для расширений производителей.
- Позволяет передавать Indoor-данные (BLE, WiFi, UWB, NFC) в уже существующей инфраструктуре РНИС (Пермь, Камчатка, МО).

**Преимущества такого подхода**:
- Полная совместимость с существующим EGTS-парсером и БНСО.
- Нет необходимости менять транспортный уровень.
- Легко комбинировать с GPS (`EGTS_SR_POS_DATA` SRT 16).

**Ограничения**:
- Не является прямой реализацией Air Interface из ISO/IEC 24730 (радио-уровень остаётся на BLE/WiFi/UWB).
- Фокус на **транспортном/прикладном уровне** (передача данных по EGTS).

## Рекомендации по развитию

1. **Соответствие API**: Реализовать функции в соответствии с ISO/IEC 24730-1 (например, в Cloud Function или мобильном приложении).
2. **UWB-поддержка**: Добавить SRT для данных из 24730-61/62.
3. **Документация**: Ссылки на ГОСТ Р ИСО/МЭК 24730 в ТЗ.
4. **Тестирование**: Генерация пакетов, совместимых с требованиями стандарта к точности и частоте обновлений.
5. **Интеграция**: В публичном портале РНИС показывать Indoor-зоны поверх карты.
